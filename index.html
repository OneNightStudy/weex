weex创建APP 简略教程

这里只针对windows系统环境下，生成android的APP

其他系统或者ios原理相同,可自行百度

需要配置太多的环境，很有可能某一步就不能通过，请不要灰心，针对出现的问题百度加以解决

第一步：安装weex(包括node,weex等)

1.下载node.js，配置环境变量(如果失败，请百度node安装)

node下载
傻瓜式一直点击下一步，安装完成后
在Windows环境下，请打开命令提示符(win+R,输出cmd回车)，然后输入node -v，如果安装正常，你应该看到这样的版本信息：
C:\Users\Administrator>node -v
v10.15.1
说明node已经安装完成，npm(node的包管理工具)也顺带装好了


2.直接使用 npm 来安装 weex-toolkit(如果失败，请百度weex安装)

在当前命令提示符输入npm install -g weex-toolkit,回车等待安装
完成后输入weex -v可查看当前版本
C:\Users\Administrator>weex -v
v1.3.11
第二步：安装 Java JDK,已安装的请跳过

1.在安装 Android Studio 之前，需安装好 Java JDK。点我跳转下载(如果失败，请百度Java JDK安装)


2.下载之后，就是傻瓜式的安装了，一直下一步,复制你下面的安装路径，安装完成后，配置环境变量会用到。


3.环境变量的配置：右击我的电脑->属性->高级->环境变量->系统变量，单击[新建]添加环境变量 


              
3.1:新建JAVA_HOME

              
变量名JAVA_HOME

              
变量值  C:\Program Files\Java\jdk1.8.0_131你的jdk安装地址

            


              
3.2:配置CLASSPATH

              
变量名CLASSPATH

              
变量值.;%JAVA_HOME%\lib

            


              
3.3:配置PATH

              
变量名Path

              
变量值%JAVA_HOME%\bin


由于安装环境比较麻烦,请打开命令提示符(win+R,输出cmd回车),依次输入验证

java -version
javac
java

就这样java环境就搭建好了

第三步：安装android所需要的环境(包括Android Studio,SDK等),已安装的请跳过

好累啊，你自己百度吧(Android Studio安装)




良心发现，放点连接

Android Studio安装教程

注意:按照教程安装完即可，记住SDK的安装位置，配置环境要用，不用新建项目

安装完Android Studio后，SDK配置环境教程

第四步：创建项目,生成APK

来到这里，所有的环境配置都已经完成，可以开始实干兴邦

在桌面新建文件夹"weexApp"，打开文件夹后,"shift+右键"，打开命令行窗口

在命令提示符输入weex create awesome-project

有输入的地方直接回车，然后下载，完成后就初始化成功"awesome-app"项目了

1.进入刚刚创建的文件夹
cd awesome-app
2.安装依赖，可能需要几分钟
npm install
3.安装应用模板
weex platform add android
4.weex打包成js
npm run build
5.打包成为签名的apk
npm run pack:android
6.签名（签名比较复杂，我往简单处理）
6.1 输入命令keytool -genkeypair -alias icesslkey -keyalg RSA -validity 3650 -keystore icekeystore.jks
6.2 提示输入两次密码,都输入android
6.3 其余信息可以直接回车,确认时输入y再回车
6.4 弹出建议时输入keytool -importkeystore -srckeystore icekeystore.jks -destkeystore icekeystore.jks -deststoretype pkcs12
6.5 再次输入确认密码android

6.6 最后输入打包命令jarsigner -verbose -keystore icekeystore.jks -signedjar myApp.apk release/android/app-release-unsigned.apk icesslkey
第一次打包会下载比较多内容，耗时较长






至此，apk已经打包完毕
